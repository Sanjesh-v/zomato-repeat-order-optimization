# A/B Testing Plan: Smart Reorder Hub

## Objective

Increase Repeat Order Rate by reducing decision fatigue.

---

## Hypothesis

Users exposed to a Smart Reorder Hub will place repeat orders more frequently than users using the current homepage.

---

## Experiment Design

### Control Group

Current Zomato Homepage

### Treatment Group

Homepage with Smart Reorder Hub displaying:

- Last 5 Orders
- Favorite Restaurants
- One-Click Reorder

---

## Primary Metric

Repeat Order Rate

Formula:

Users with 2+ Orders / Total Ordering Users

---

## Secondary Metrics

- Orders Per User
- Revenue Per User
- Cart Conversion Rate
- Session Duration

---

## Guardrail Metrics

- App Crash Rate
- Checkout Failure Rate
- Customer Support Tickets

---

## Success Criteria

- +5% Repeat Order Rate
- 95% Statistical Confidence
- No negative impact on checkout conversion

---

## Rollout Plan

Phase 1: 10% Users

Phase 2: 50% Users

Phase 3: 100% Users
